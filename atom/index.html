<p>erlang中函数调用分为两种：</p> <ol> <li>remote call/external function call: <code>m:f(e1, ..., eN)</code></li> <li>local call: <code>f(e1, ..., eN)</code></li> </ol> <p>remote call 和 local call的主要差别在热更新时。erlang运行时对module可以维护 两个版本的代码:<code>current</code>和 <code>old</code>。老版本和新版本可以同时运行：remote call总是使用 <code>current</code>版本，local call可能运行<code>old</code>版本。当有更新的版本代码加载时，运行<code>old</code> 代码的进程会终止。可以通过调用remote call，使进程切换到 <code>current</code> 版本。</p> <pre class="highlight"><code>-module(m). -export([loop/0]). loop() -&gt; receive code_switch -&gt; m:loop(); Msg -&gt; do_something, loop() end.</code></pre> <p>这种差别也体现在spawn函数调用时，<code>spawn(M, F, A)</code> 使用<code>current</code>代码，而 <code>spawn(Fun)</code> 中 <code>Fun</code> 是local call，不能进行热更新。 </p> <p>但是存在这样一个问题：当进行热更新时，怎么保证运行老版本代码的进程不死掉呢？</p> <h4>--- Ref ---</h4> <p>[1] <a href="http://erlang.org/doc/reference_manual/expressions.html#id78068">Function Calls</a></p> <p>[2] <a
